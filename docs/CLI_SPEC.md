# agstage CLI Specification

This document defines the complete CLI contract for agstage v2: command syntax, flags, JSON schemas, exit codes, and example outputs. AI agents program against this specification.

All commands output JSON to stdout. Errors output JSON to stdout with a non-zero exit code. Exit codes are always set; see [Exit Codes](#exit-codes).

---

## Global Flags

These flags apply to all commands.

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--repo PATH` | string | CWD | Repository path (discovered via `.git` if not specified) |
| `--context N` | u32 | 3 | Context lines for diff generation (minimum: 1, enforced at runtime) |

### Global Flag Notes

- `--repo` discovery: if not provided, walks up from CWD looking for `.git` directory.
- `--context`: controls the number of context lines in diffs generated by the diff engine (via `git2::DiffOptions::context_lines`). Affects all commands that compute diffs: `scan`, `stage`, `unstage`, `status`. Minimum value is 1 (values below 1 are clamped).

## Selection Syntax: Auto-Detection

agstage v2 uses flat positional arguments with auto-detection instead of explicit `--file`/`--hunk`/`--lines` flags. The detection rules, applied in order:

1. **Lines**: If the string contains `:` and the character immediately after the **last** `:` is an ASCII digit, parse as `path:START-END[,START-END,...]`. Line ranges are 1-indexed and inclusive.
2. **Hunk**: If the string is exactly 12 hexadecimal characters, treat as a hunk ID.
3. **File**: Everything else is treated as a file path.

**Examples**:

| Input | Detected As | Parsed |
|-------|-------------|--------|
| `src/main.rs` | File | `{ path: "src/main.rs" }` |
| `abc123def456` | Hunk | `{ hunk_id: "abc123def456" }` |
| `src/main.rs:10-20` | Lines | `{ path: "src/main.rs", ranges: [{start: 10, end: 20}] }` |
| `src/lib.rs:1-5,10-15` | Lines | `{ path: "src/lib.rs", ranges: [{start: 1, end: 5}, {start: 10, end: 15}] }` |
| `abc123def45` | File | 11 chars, not exactly 12 hex |
| `C:\Users\file.rs` | File | Last `:` followed by `\`, not a digit |
| `C:\Users\file.rs:5-10` | Lines | Last `:` followed by `5` |

## Commands

### agstage scan

Discover all unstaged changes in the working tree, including untracked (new) files. Untracked files appear with `status: Added`. Files matched by `.gitignore` are excluded.

**Syntax**:
```
agstage scan [FILES...] [--full]
```

**Arguments**:

| Argument | Type | Description |
|----------|------|-------------|
| `FILES` | positional, repeatable | Files to scan (omit for all changed files) |

**Flags**:

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--full` | flag | false | Include full diff line content and checksums |

**Default Output Schema: CompactScanResult**

By default, `agstage scan` returns compact output with hunk metadata but no diff line content:

```typescript
{
  "files": CompactFileInfo[],
  "summary": ScanSummary
}
```

**CompactFileInfo**:
```typescript
{
  "path": string,                    // relative path from repo root
  "status": FileStatus,             // Added | Modified | Deleted | Renamed
  "is_binary": boolean,             // true if file contains binary content
  "hunks": CompactHunkInfo[],
  "hunks_count": number,            // number of hunks in this file
  "lines_added": number,            // total lines added across all hunks
  "lines_deleted": number           // total lines deleted across all hunks
}
```

**CompactHunkInfo**:
```typescript
{
  "hunk_id": string,                 // content-based ID, usable with `stage`
  "header": string,                  // raw @@ header, e.g., "@@ -10,3 +10,5 @@ fn main()"
  "old_start": number,               // starting line in original file (1-indexed)
  "old_lines": number,               // number of lines in original file
  "new_start": number,               // starting line in new file (1-indexed)
  "new_lines": number,               // number of lines in new file
  "additions": number,               // count of added lines in this hunk
  "deletions": number                // count of deleted lines in this hunk
}
```

**Compact Example Output**:
```json
{
  "files": [
    {
      "path": "src/main.rs",
      "status": { "type": "Modified" },
      "is_binary": false,
      "hunks": [
        {
          "hunk_id": "abc123def456",
          "header": "@@ -10,3 +10,5 @@ fn main()",
          "old_start": 10,
          "old_lines": 3,
          "new_start": 10,
          "new_lines": 5,
          "additions": 2,
          "deletions": 0
        }
      ],
      "hunks_count": 1,
      "lines_added": 2,
      "lines_deleted": 0
    }
  ],
  "summary": {
    "total_files": 1,
    "total_hunks": 1,
    "added": 0,
    "modified": 1,
    "deleted": 0,
    "renamed": 0,
    "binary": 0
  }
}
```

**Full Output Schema (`--full`): ScanResult**

```typescript
{
  "files": FileInfo[],
  "summary": ScanSummary
}
```

**FileInfo**:
```typescript
{
  "path": string,                    // relative path from repo root
  "status": FileStatus,             // Added | Modified | Deleted | Renamed
  "file_checksum": string,          // SHA-256 hex of working-tree file content
  "is_binary": boolean,             // true if file contains binary content
  "hunks": HunkInfo[]
}
```

**FileStatus** (tagged union):
```typescript
{ "type": "Added" }
| { "type": "Modified" }
| { "type": "Deleted" }
| { "type": "Renamed", "old_path": string }
```

**HunkInfo**:
```typescript
{
  "hunk_id": string,                 // content-based ID: sha256(path:old_start:new_start:lines)[0..12]
  "old_start": number,
  "old_lines": number,
  "new_start": number,
  "new_lines": number,
  "header": string,                  // raw @@ header
  "lines": DiffLineInfo[],
  "checksum": string                 // SHA-256 hex of hunk content
}
```

**DiffLineInfo**:
```typescript
{
  "line_number": number,             // 1-indexed; new file for additions/context, old file for deletions
  "origin": "Context" | "Addition" | "Deletion",
  "content": string                  // line content (without leading +/- prefix)
}
```

**ScanSummary**:
```typescript
{
  "total_files": number,
  "total_hunks": number,
  "added": number,
  "modified": number,
  "deleted": number,
  "renamed": number,
  "binary": number
}
```

**Full Example Output**:
```json
{
  "files": [
    {
      "path": "src/main.rs",
      "status": { "type": "Modified" },
      "file_checksum": "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2",
      "is_binary": false,
      "hunks": [
        {
          "hunk_id": "abc123def456",
          "old_start": 10,
          "old_lines": 3,
          "new_start": 10,
          "new_lines": 5,
          "header": "@@ -10,3 +10,5 @@",
          "lines": [
            { "line_number": 10, "origin": "Context", "content": "fn main() {" },
            { "line_number": 11, "origin": "Addition", "content": "    println!(\"Hello\");" },
            { "line_number": 12, "origin": "Addition", "content": "    println!(\"World\");" },
            { "line_number": 13, "origin": "Context", "content": "}" }
          ],
          "checksum": "f7e6d5c4b3a2f7e6d5c4b3a2f7e6d5c4b3a2f7e6d5c4b3a2f7e6d5c4b3a2f7e6"
        }
      ]
    }
  ],
  "summary": {
    "total_files": 1,
    "total_hunks": 1,
    "added": 0,
    "modified": 1,
    "deleted": 0,
    "renamed": 0,
    "binary": 0
  }
}
```

**Exit Codes**:
- `0`: Success
- `1`: No changes detected (repository has no unstaged changes)
- `4`: Internal error (git2 error, IO error)

### agstage stage

Stage selected changes into the index. Always atomic: on first failure, the index is restored from backup and the operation returns an error.

**Syntax**:
```
agstage stage SELECTION... [--exclude SELECTION]... [--dry-run]
```

**Arguments**:

| Argument | Type | Description |
|----------|------|-------------|
| `SELECTION` | positional, repeatable | Selections to stage (auto-detected: file path, 12-hex hunk ID, path:range) |

**Flags**:

| Flag | Type | Repeatable | Description |
|------|------|-----------|-------------|
| `--exclude` | string | yes | Exclude selections (same auto-detect syntax as positional args) |
| `--dry-run` | flag | no | Validate and report without modifying the index |

**Output Schema: StageResult**

```typescript
{
  "status": "Ok" | "DryRun",
  "succeeded": StagedItem[],
  "failed": FailedItem[],
  "warnings": string[],
  "backup_id": string
}
```

**StagedItem**:
```typescript
{
  "selection": string,              // the selection that was staged (e.g., "src/main.rs" or "abc123def456")
  "lines_staged": number            // number of lines successfully staged
}
```

**FailedItem**:
```typescript
{
  "selection": string,              // the selection that failed
  "reason": string,                 // machine-readable reason
  "suggestion": string              // human/agent-readable guidance for recovery
}
```

**Example Output** (successful):
```json
{
  "status": "Ok",
  "succeeded": [
    {
      "selection": "src/main.rs",
      "lines_staged": 7
    },
    {
      "selection": "abc123def456",
      "lines_staged": 3
    }
  ],
  "failed": [],
  "warnings": [],
  "backup_id": "backup-20260211T143044-6ba7b810"
}
```

**Example Output** (dry-run):
```json
{
  "status": "DryRun",
  "succeeded": [
    {
      "selection": "src/main.rs",
      "lines_staged": 7
    }
  ],
  "failed": [],
  "warnings": [],
  "backup_id": "backup-20260211T143200-dryrun12"
}
```

**Exit Codes**:
- `0`: Success (status = "Ok" or "DryRun")
- `1`: No effect (nothing matched; selection was empty)
- `2`: User error (invalid syntax, invalid line range, unknown hunk ID, granular on binary/whole-file)
- `3`: Conflict/failure (stale scan, index locked, staging write failed)
- `4`: Internal error (git2 error, IO error)

### agstage unstage

Remove selected changes from the index while preserving them in the working tree. Opposite of `agstage stage`. Always atomic.

**Syntax**:
```
agstage unstage SELECTION... [--exclude SELECTION]... [--dry-run]
```

**Arguments**:

| Argument | Type | Description |
|----------|------|-------------|
| `SELECTION` | positional, repeatable | Selections to unstage (auto-detected: file path, 12-hex hunk ID, path:range) |

**Flags**:

| Flag | Type | Repeatable | Description |
|------|------|-----------|-------------|
| `--exclude` | string | yes | Exclude selections (same auto-detect syntax) |
| `--dry-run` | flag | no | Validate and report without modifying the index |

**Output Schema: StageResult**

Same as `agstage stage` (reuses `StageResult` schema).

**Example Output** (successful):
```json
{
  "status": "Ok",
  "succeeded": [
    {
      "selection": "abc123def456",
      "lines_staged": 3
    }
  ],
  "failed": [],
  "warnings": [],
  "backup_id": "backup-20260211T143320-c7d8e9f0"
}
```

**Exit Codes**:
- `0`: Success
- `1`: No effect (nothing matched)
- `2`: User error (invalid syntax, unknown hunk ID)
- `3`: Conflict/failure (index locked, staging write failed)
- `4`: Internal error (git2 error, IO error)

### agstage status

Show currently staged changes (HEAD vs. index).

**Syntax**:
```
agstage status
```

**Flags**: None (inherits global flags).

**Output Schema: StatusReport**

```typescript
{
  "staged_files": StagedFileInfo[],
  "summary": StatusSummary
}
```

**StagedFileInfo**:
```typescript
{
  "path": string,                    // relative path from repo root
  "status": FileStatus,             // Added | Modified | Deleted | Renamed
  "lines_added": number,
  "lines_deleted": number
}
```

**StatusSummary**:
```typescript
{
  "total_files": number,
  "total_additions": number,
  "total_deletions": number
}
```

**Example Output**:
```json
{
  "staged_files": [
    {
      "path": "src/main.rs",
      "status": { "type": "Modified" },
      "lines_added": 10,
      "lines_deleted": 2
    },
    {
      "path": "new_feature.rs",
      "status": { "type": "Added" },
      "lines_added": 47,
      "lines_deleted": 0
    }
  ],
  "summary": {
    "total_files": 2,
    "total_additions": 57,
    "total_deletions": 2
  }
}
```

**Exit Codes**:
- `0`: Success (even if no staged changes; summary will have 0 files)
- `4`: Internal error (git2 error, IO error)

### agstage commit

Create a git commit from staged changes. Uses author name and email from git config.

**Syntax**:
```
agstage commit -m MESSAGE
```

**Flags**:

| Flag | Type | Description |
|------|------|-------------|
| `-m, --message` | string (required) | Commit message |

**Output Schema: CommitResult**

```typescript
{
  "commit_hash": string,     // full 40-char hex SHA-1
  "message": string,
  "author": string,          // "Name <email>" from git config
  "files_changed": number,
  "insertions": number,
  "deletions": number
}
```

**Example Output**:
```json
{
  "commit_hash": "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2",
  "message": "feat: add new feature",
  "author": "Jane Doe <jane@example.com>",
  "files_changed": 3,
  "insertions": 42,
  "deletions": 7
}
```

**Exit Codes**:
- `0`: Success
- `1`: No effect (nothing staged)
- `4`: Internal error (git2 error)

## Exit Codes

All commands return one of these exit codes:

| Code | Meaning | When | Examples |
|------|---------|------|----------|
| `0` | Success | Operation completed | Staged/unstaged as requested; scan found changes; status reported |
| `1` | No effect | Nothing to do | No unstaged changes; selection matched no hunks; nothing staged for commit |
| `2` | User error | Invalid input | Invalid selection syntax; unknown hunk ID; invalid line range; granular on binary file; granular on added/deleted/renamed file |
| `3` | Conflict | Operational failure (retryable) | Stale scan (file changed since scan); index locked by another process; staging write failed |
| `4` | Internal error | Code/system error | git2 error; IO error; JSON serialization error |

---

## Error JSON Schema

When a command fails (exit code >= 1 for errors), it outputs:

```typescript
{
  "error": string,                   // human-readable error message
  "exit_code": number                // the exit code (1, 2, 3, or 4)
}
```

**Example**:
```json
{
  "error": "stale scan detected for src/main.rs: file has changed since last scan",
  "exit_code": 3
}
```

## Notes for Agents

1. **Compact output is default**: `agstage scan` returns compact output without diff line content. This keeps output small for LLM context windows. Use `--full` when you need per-line content. Hunk IDs in compact output are identical to `--full` and work directly with `stage`.

2. **Hunk IDs are stable**: Once you get a hunk ID from `scan`, you can use it immediately in `stage`/`unstage` as long as the file has not changed. If the file is modified between scan and stage, the operation will fail with exit code 3 (`stale_scan`) suggesting a re-scan.

3. **Positional args with auto-detection**: No `--file`/`--hunk`/`--lines` flags needed. Just pass the selection directly: `agstage stage src/main.rs`, `agstage stage abc123def456`, `agstage stage src/main.rs:10-20`.

4. **Always atomic**: Operations stop on first failure and restore the index from backup. There is no partial success state.

5. **Always backed up**: Every stage/unstage operation creates an index backup in `.git/agstage/backups/` before making changes. The `backup_id` is returned in the result.

6. **Dry-run before committing**: Use `--dry-run` to validate a staging operation before modifying the index.

7. **Incremental staging**: You can call `stage` multiple times with different selections in sequence. Each call is independently atomic with its own backup.

8. **Line-range syntax**: Always use 1-indexed, inclusive line numbers. `src/main.rs:1-10,20-30` stages lines 1-10 and lines 20-30.

9. **Binary file constraints**: Binary files can only be staged at file level. Attempting hunk or line-level selections on a binary file returns exit code 2 (`BinaryFileGranular`).

10. **Whole-file constraints**: Added, deleted, and renamed files can only be staged at file level. Attempting hunk or line-level selections on such files returns exit code 2 (`GranularOnWholeFile`).

11. **Context lines**: The `--context N` flag affects hunk boundaries in scan output. Larger values may merge nearby hunks; smaller values keep them separate. Default is 3.

## Example Workflows

### Workflow 1: Stage a single file

```bash
# Scan to see what changed
agstage scan

# Stage the entire file
agstage stage src/main.rs

# Verify
agstage status

# Commit
agstage commit -m "feat: update main entry point"
```

### Workflow 2: Stage a specific hunk

```bash
# Scan to get hunk IDs
agstage scan

# Stage just that hunk
agstage stage abc123def456

# Verify
agstage status
```

### Workflow 3: Stage specific line ranges

```bash
# Scan to understand the structure
agstage scan

# Stage lines 10-20 and 50-60
agstage stage src/main.rs:10-20 src/utils.rs:50-60

# Verify
agstage status
```

### Workflow 4: Dry-run then execute

```bash
# Validate without modifying
agstage stage src/main.rs --dry-run

# If successful (status: "DryRun"), execute for real
agstage stage src/main.rs
```

### Workflow 5: Multiple selections with exclusions

```bash
# Stage entire file but exclude a specific hunk
agstage stage src/main.rs --exclude abc123def456
```

### Workflow 6: Full scan for code review

```bash
# Get full diff line content
agstage scan --full

# Filter to specific files
agstage scan src/main.rs src/lib.rs --full
```

### Workflow 7: Unstage a change

```bash
# Unstage a specific hunk
agstage unstage abc123def456

# Unstage an entire file
agstage unstage src/main.rs

# Verify
agstage status
```

- **Extensions**: New fields may be added to JSON output in minor revisions (agents should ignore unknown fields).
